# 需求文档：YouTube 字幕对话稿人物标注工具

## 1. 项目背景

YouTube 自动字幕通常只有连续文本或时间戳文本，缺少说话人标识。用户在整理访谈、课程、播客、视频教程时，需要将字幕整理成带人物标注的对话稿，便于阅读、引用和后续处理。

由于完整字幕可能长达 3-4 小时，直接一次性丢给大模型容易超过上下文限制，也容易造成角色识别混乱。因此需要先对字幕进行清洗、分块，再让大模型逐块识别人物，最后统一合并。

## 2. 使用对象

- 视频学习者：整理课程、访谈、播客内容。
- 内容创作者：将视频字幕整理为文章素材。
- 研究人员：分析访谈或会议文本。
- 普通用户：把长字幕转成清晰对话稿。

## 3. 功能目标

第一版需要完成：

- 支持粘贴英文字幕文本。
- 自动清洗无用符号、重复空行、异常换行。
- 按自然句子和字数限制分块。
- 每个分块保留前后重叠上下文。
- 调用大模型识别说话人。
- 输出结构化 JSON。
- 合并多个分块结果。
- 统一人物名称。
- 导出 Markdown 对话稿。

## 4. 输入内容

支持输入：

- YouTube 自动字幕文本。
- 带时间戳的字幕文本。
- 普通英文转写文本。

示例：

```text
00:00 Hello everyone, welcome back.
00:04 Today we have Nick with us.
00:08 Thanks for having me.
```

## 5. 输出内容

输出带人物标注的 Markdown：

```md
# 对话稿

## 人物表

- Host：主持人
- Nick：嘉宾

## 正文

**Host：** Hello everyone, welcome back.

**Nick：** Thanks for having me.
```

## 6. 正常流程

1. 用户粘贴字幕文本。
2. 系统清洗字幕。
3. 系统按句子和长度进行分块。
4. 系统为每块加入前后重叠上下文。
5. 大模型识别每块人物和对话内容。
6. 系统汇总所有分块结果。
7. 系统生成全局人物表。
8. 系统统一人物名称。
9. 系统导出 Markdown 对话稿。

## 7. 异常流程

- 输入为空：提示用户输入字幕文本。
- 字幕过短：直接作为单块处理。
- 字幕过长：自动分块处理。
- 模型返回 JSON 解析失败：重试一次，仍失败则记录错误块。
- 人物识别置信度低：标记为 `Unknown Speaker`。
- 分块合并冲突：保留原文，并在人物表中标记待确认。
- API 调用失败：保留当前处理进度，提示失败原因。

## 8. 验收标准

第一版完成后必须满足：

- 能输入一段字幕文本。
- 能自动清洗明显无用内容。
- 能按句子边界分块，不硬切句子。
- 能对每块返回人物标注 JSON。
- 能合并多个分块结果。
- 能输出 Markdown 对话稿。
- 人物名称前后基本一致。
- 不强行编造原字幕中不存在的内容。
