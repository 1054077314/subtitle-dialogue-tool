# CLAUDE.md

## 关于项目

这是一个 YouTube 字幕对话稿人物标注工具。目标是把长字幕清洗、分块、识别人名，并导出带人物标注的 Markdown 对话稿。

第一版只做最小闭环：

```text
字幕文本输入 → 清洗 → 分块 → AI 识别人物 → 合并 → 导出 Markdown
```

## 沟通方式

- 默认中文。
- 结论先行。
- 不要夸我的想法。
- 方案有问题直接指出。
- 代码、命令、变量名用英文。

## 开发原则

- 先结构稳定，再加功能。
- 一次只做一个模块。
- 一次只改少量文件。
- 不要重写整个项目。
- 不要做无关优化。
- 不要引入新框架，除非我明确要求。

## 项目边界

第一版不做：

- 用户登录。
- 数据库。
- YouTube 自动下载。
- 视频播放器。
- 多用户系统。
- 云端部署。
- 复杂前端 UI。

## 模块规则

- subtitle-cleaner 只负责清洗，不负责分块。
- subtitle-splitter 只负责分块，不调用大模型。
- speaker-analyzer 只负责单块人物识别。
- result-merger 只负责合并分块结果。
- speaker-normalizer 只负责统一人物名称。
- markdown-exporter 只负责导出 Markdown。

## 禁止事项

- 不要删除现有功能。
- 不要修改已确认的数据格式。
- 不要把所有逻辑写进一个文件。
- 不要让模型直接输出最终文本而跳过 JSON。
- 不要硬切 8000 字，必须尽量按句子边界切。
- 不要编造字幕中不存在的内容。
- 不要把 Unknown Speaker 强行合并成人名。

## 验收要求

完成每个任务后只汇报：

- 修改了哪些文件。
- 当前模块是否能单独运行。
- 是否改变了数据格式。
- 是否影响其他模块。
- 是否通过 smoke 验收。
