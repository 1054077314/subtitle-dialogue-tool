# 开发计划文档

## 开发原则

- 一次只做一个模块。
- 每个模块必须能单独运行和验证。
- 不做完整系统大爆炸式实现。
- 第一版先做命令行或简单页面。
- 每完成 2-3 个任务，检查一次结构。

## Task 1：字幕输入与清洗

目标：

- 支持输入字幕文本。
- 清理多余空行和异常换行。
- 输出清洗后的文本。

允许修改：

- subtitle-input 相关文件
- subtitle-cleaner 相关文件

验收：

- 输入空文本能提示错误。
- 输入普通字幕能正常清洗。
- 清洗后不丢失原文内容。

---

## Task 2：字幕分块

目标：

- 按句子边界分块。
- 每块控制在指定长度。
- 添加前后重叠上下文。

验收：

- 不从句子中间硬切。
- 长字幕能拆成多个 chunk。
- 每个 chunk 有 chunk_id。
- 每个 chunk 带 context_before / context_after。

---

## Task 3：单块 AI 人物识别

目标：

- 单个 chunk 调用大模型。
- 返回人物列表和对话列表。
- 输出 JSON。

验收：

- JSON 可解析。
- 每条 dialogue 有 speaker_id、speaker_name、text、confidence。
- 不确定人物标为 Unknown Speaker。
- 不编造原文内容。

---

## Task 4：多块批量处理

目标：

- 遍历所有 chunk。
- 保存每块结果。
- 失败块单独记录。

验收：

- 多个 chunk 能依次处理。
- 某块失败不影响全部流程。
- 有错误日志。

---

## Task 5：人物统一

目标：

- 汇总所有 people。
- 生成全局 speaker_map。
- 合并 Host / Speaker 1 等重复人物。

验收：

- 同一人物名称尽量统一。
- 低置信度人物保留待确认标记。
- 不强行合并不同人物。

---

## Task 6：导出 Markdown

目标：

- 导出人物表。
- 导出正文对话。
- 保留处理说明。

验收：

- Markdown 可直接阅读。
- 人物标注清晰。
- 顺序与原字幕一致。

---

## Task 7：简单 UI

目标：

- 输入字幕。
- 点击开始处理。
- 展示处理结果。
- 支持复制/下载 Markdown。

验收：

- 页面能打开。
- 按钮能点击。
- 处理结果能展示。
- 控制台无明显报错。
